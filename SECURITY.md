# Security Policy

## Supported versions

Only the latest main branch is actively supported.

## Reporting a vulnerability

Please report security issues privately to the maintainers instead of opening a public issue.

When reporting, include:
- affected command(s)
- reproduction steps
- impact assessment
- any known mitigations

## Sensitive data guidance

- Never commit `.env`.
- Never share session cache files.
- Avoid passing passwords directly in shell history when possible.
